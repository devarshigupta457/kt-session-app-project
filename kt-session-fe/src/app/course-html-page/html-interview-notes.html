PART 1

Fundamentals

01What HTML Is

HTML (HyperText Markup Language) is the markup language that structures web content. It's not a programming language — it describes what content is (a heading, a paragraph, a link), while CSS styles it and JavaScript adds behaviour.

The trio

HTML = structure, CSS = presentation, JavaScript = behaviour. Keeping them separated is a core best practice.

02Elements, Tags & Attributes

html
<a href="/docs" target="_blank">Read docs</a>
  • Element = start tag + content + end tag.
  • Tag = the <a> marker itself.
  • Attribute = name="value" inside the start tag (here href, target).
Void elements

Some elements have no content or closing tag: <img>, <br>, <input>, <hr>, <meta>, <link>.

04Document Structure

html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <title>Page</title>   <!-- metadata -->
  </head>
  <body> ... </body>      <!-- visible content -->
</html>
Why the doctype

<!DOCTYPE html> tells the browser to use standards mode. Without it, browsers fall back to quirks mode.

PART 2

Structure & Semantics

05Semantic HTML

Semantic elements describe their meaning, not just their look. They help accessibility, SEO, and readability.

<header> site header / logo <nav> navigation links <main> the one main content area (once per page) <section> thematic grouping <article> self-contained content (post, card) <aside> side content / related <footer> page footer
Say this

"Prefer <nav>, <main>, <article> over a wall of <div>s — semantics give screen readers landmarks to navigate by."

06Text, Links & Images

  • Headings <h1>–<h6> convey hierarchy — one <h1> per page, don't skip levels for styling.
  • Links: <a href>; use rel="noopener" with target="_blank".
  • Images: <img src alt> — always include alt text (empty alt="" for purely decorative images).

07Lists & Tables

  • <ul> unordered, <ol> ordered, <dl> description list.
  • Tables are for tabular data, not layout: <table> → <thead>/<tbody> → <tr> → <th>/<td>.
PART 3

Forms & Inputs

08Forms & Inputs

html
<form action="/submit" method="post">
  <label for="email">Email</label>
  <input id="email" name="email" type="email" required>
  <button type="submit">Send</button>
</form>
  • method: GET puts data in the URL (search, idempotent); POST sends it in the body (changes state).
  • name is the key sent to the server; id links the <label> via for for accessibility.
  • Input types: text email password number date checkbox radio file…

09Validation

The browser can validate before submit using attributes — but you must always re-validate on the server.

  • required, min/max, minlength/maxlength, pattern (regex), correct type.
Rule

Client-side validation is UX; server-side validation is security. Never trust the client.

PART 4

Metadata & Media

10Metadata & SEO

html
<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1">
<meta name="description" content="Page summary for search results">
<meta property="og:title" content="..."> <!-- social preview -->
The must-have

The viewport meta tag is what makes a page responsive on mobile. Without it, phones render at desktop width.

11Media & Embeds

  • <img> with srcset for responsive images; loading="lazy" to defer offscreen images.
  • <video> / <audio> with controls.
  • <iframe> embeds another document (maps, videos).
  • data-* attributes store custom data readable from JS via element.dataset.
PART 5

Accessibility

12Accessibility (a11y)

  • Use semantic elements — they come with built-in roles.
  • Every image needs alt; every input needs a <label>.
  • Use ARIA only to fill gaps semantics can't (e.g. aria-label, aria-expanded, role) — "no ARIA is better than bad ARIA."
  • Ensure keyboard operability and visible focus; maintain colour contrast.
PART 6

Quick-Fire Q&A

Tap a question to reveal the answer.

>Is HTML a programming language?

No — it's a markup language for structuring content. It has no logic, loops, or variables.

>Difference between id and class?

id is unique per page and used for one element; class is reusable across many. CSS specificity: id > class.

>Why use semantic HTML?

Better accessibility (screen-reader landmarks), better SEO, and clearer, more maintainable markup than nested divs.

>What does the viewport meta tag do?

It tells mobile browsers to match the layout width to the device width, enabling responsive design.

>GET vs POST on a form?

GET appends data to the URL and is for safe, idempotent reads; POST sends data in the request body and is for actions that change state.

>Why is alt text important?

It describes images for screen-reader users and shows when an image fails to load; also helps SEO.

>What are void (empty) elements?

Elements with no content or closing tag, like <img>, <br>, <input>, <meta>.

>What is the DOM?

The Document Object Model — the browser's in-memory tree representation of the HTML that JavaScript can read and modify.

Final tips for the interview

  1. Lead with semantic, accessible markup — it signals maturity.
  2. Know the document structure and why the doctype and viewport matter.
  3. Explain the HTML/CSS/JS separation of concerns.
  4. Be precise about id vs class and block vs inline.
  5. Mention alt text and labels whenever forms or images come up.
  6. Tie HTML to the DOM — interviewers bridge into JS from here.
