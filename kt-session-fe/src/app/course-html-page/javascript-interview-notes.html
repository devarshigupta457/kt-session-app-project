PART 1

Fundamentals

01var, let & const

varletconst
ScopeFunctionBlockBlock
ReassignYesYesNo
HoistingYes → undefinedHoisted, in TDZHoisted, in TDZ
Rule

Default to const; use let when you must reassign; avoid var. const stops reassignment but the object it points to can still mutate.

02Types & Coercion

Seven primitives: string number boolean null undefined symbol bigint. Everything else is an object (arrays, functions included).

js
typeof "hi"     // "string"
typeof null     // "object"  (historical bug)
typeof () => {}  // "function"

JS coerces types in loose contexts ("5" + 1 is "51", "5" - 1 is 4). Prefer explicit conversion.

03Hoisting & Scope

Hoisting: declarations are conceptually moved to the top of their scope. var is initialised to undefined; let/const exist but are in the Temporal Dead Zone until assigned. Function declarations are fully hoisted (callable before their line); function expressions are not.

04== vs ===

js
0 == ""       // true  (coercion)
0 == false    // true
0 === ""      // false (no coercion)
null == undefined  // true
Use ===

== coerces types before comparing; === checks type and value. Always use === unless you have a specific reason.

PART 2

Functions

05Closures

A closure is a function that remembers variables from the scope where it was created, even after that scope has returned.

js
function counter() {
  let count = 0;
  return () => ++count;   // closes over `count`
}
const next = counter();
next(); next();  // 1, then 2 -- state is preserved & private
Why it matters

Closures power data privacy, callbacks, and hooks. A classic bug: var in a loop shares one binding — use let to capture per-iteration.

06this & Binding

this depends on how a function is called, not where it's defined.

  • Method call obj.fn() → this = obj.
  • Plain call fn() → this = undefined (strict) or global.
  • Explicit: fn.call(x), fn.apply(x, args), fn.bind(x).
  • new Fn() → this = the new object.

07Arrow vs Regular Functions

Arrow functions don't have their own this — they capture it lexically from the surrounding scope. They also have no arguments and can't be used as constructors.

js
const obj = {
  items: [1,2],
  show() { // regular: `this` is obj
    this.items.forEach(i => console.log(this.items, i));
    // arrow keeps `this` = obj -- no need for `self = this`
  }
};
PART 3

Objects & Prototypes

08Prototypes & Classes

JS uses prototypal inheritance: every object links to a prototype object; property lookups walk up the prototype chain. class is syntactic sugar over this.

js
class Animal {
  constructor(name) { this.name = name; }
  speak() { return `${this.name} makes a sound`; }
}
class Dog extends Animal {
  speak() { return `${this.name} barks`; } // override
}

09Destructuring & Spread

js
const { name, age = 0 } = user;      // object destructure + default
const [first, ...rest] = list;       // array destructure + rest
const merged = { ...a, ...b };        // spread / shallow copy
fn(...args);                            // spread into arguments
PART 4

Asynchronous JavaScript

10Promises & async/await

A Promise represents a future value in one of three states: pending, fulfilled, or rejected. async/await is cleaner syntax over .then().

js
async function load() {
  try {
    const res  = await fetch("/api/user");
    const user = await res.json();
    return user;
  } catch (e) { console.error(e); }
}
Callback hell → Promises

Promises flatten nested callbacks; await makes async code read top-to-bottom. Use Promise.all([...]) to run tasks concurrently.

11The Event Loop

JS is single-threaded. The event loop lets it stay non-blocking:

Call Stack runs sync code | async op (setTimeout, fetch) handed to Web APIs v Microtask queue (Promises) ← drained FIRST, fully Macrotask queue (setTimeout, events) ^ event loop pushes the next callback when the stack is empty
Key ordering

After each task, all microtasks (Promise callbacks) run before the next macrotask. That's why a resolved Promise's .then fires before a setTimeout(fn, 0).

PART 5

Modern JS & the DOM

12Array Methods

js
nums.map(n => n * 2)          // transform -> new array
nums.filter(n => n > 0)        // keep matches -> new array
nums.reduce((sum, n) => sum + n, 0) // fold to one value
nums.find(n => n === 5)      // first match or undefined
nums.forEach(n => log(n))     // side effects, returns nothing
map vs forEach

map returns a new array; forEach returns undefined. Reach for map/filter/reduce to keep code declarative and immutable.

13DOM & Events

js
const btn = document.querySelector(".btn");
btn.addEventListener("click", e => {
  e.preventDefault();
  console.log(e.target);
});
Event delegation

Attach one listener to a parent and read event.target, instead of one listener per child. Works for dynamically added elements and uses less memory — it relies on event bubbling.

PART 6

Quick-Fire Q&A

Tap a question to reveal the answer.

>var vs let vs const?

var is function-scoped and hoisted to undefined; let/const are block-scoped with a temporal dead zone. const can't be reassigned.

>What is a closure?

A function bundled with references to its surrounding scope, so it keeps access to those variables after the outer function returns.

>== vs ===?

== compares after type coercion; === compares type and value with no coercion. Prefer ===.

>How does this work?

It's determined by the call site: method call = the object, plain call = undefined/global, arrow = lexical this, new = the new instance, or set explicitly via call/apply/bind.

>Explain the event loop.

JS runs sync code on the call stack; async callbacks wait in queues. When the stack empties, the loop runs all microtasks (Promises), then one macrotask, and repeats — keeping it non-blocking despite one thread.

>Microtask vs macrotask?

Promise callbacks are microtasks and drain fully after each task; setTimeout/DOM events are macrotasks. Microtasks always run before the next macrotask.

>map vs forEach?

map returns a new transformed array; forEach just iterates and returns undefined.

>What is hoisting?

Declarations are processed before execution. var becomes undefined; function declarations are fully available; let/const throw if used before assignment.

>What is event delegation?

Handling events on a parent element via bubbling and event.target, instead of binding listeners to each child.

>Prototypal inheritance?

Objects inherit by linking to a prototype object; lookups walk the prototype chain. class is sugar over this mechanism.

Final tips for the interview

  1. Be crisp on closures, this, and the event loop — the top-three JS asks.
  2. Always justify === over == and know the coercion quirks.
  3. Explain microtasks vs macrotasks with a setTimeout-vs-Promise example.
  4. Prefer immutable, declarative array methods over manual loops.
  5. Know async/await and Promise.all for concurrency.
  6. Bridge to the DOM: querySelector, listeners, and event delegation.
