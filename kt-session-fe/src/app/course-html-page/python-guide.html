PART 1

Fundamentals

01The Language

Python is a dynamically typed, interpreted, high-level language. Your source is compiled to bytecode, executed by the Python Virtual Machine (CPython is the reference implementation). Everything — numbers, functions, classes — is an object.

Dynamic + strong typing

Variables have no fixed type (dynamic), but Python won't silently coerce unrelated types like 1 + "a" (strong). Types are checked at runtime.

02Mutable vs Immutable

ImmutableMutable
int, float, str, tuple, frozenset, bool, byteslist, dict, set, bytearray, most objects
Why it matters

Immutable objects are hashable (usable as dict keys / set members) and safe to share. Mutating a shared mutable object affects every reference to it — the source of many bugs.

03is vs ==

python
a = [1, 2]; b = [1, 2]
a == b   # True  -> equal VALUES
a is b   # False -> different OBJECTS (identity)
x = None
x is None  # the correct way to check for None

== compares values (via __eq__); is compares identity (same object in memory). Use is only for singletons like None.

04Names & Scope (LEGB)

Python resolves names by the LEGB rule: Local → Enclosing → Global → Built-in. Assigning to a name inside a function makes it local unless declared global/nonlocal.

Assignment vs reference

Variables are names bound to objects, not boxes holding values. Assignment rebinds a name; it doesn't copy.

06Comprehensions

python
squares = [x*x for x in range(10) if x % 2 == 0]   # list
by_id   = {u.id: u for u in users}                  # dict
unique  = {w.lower() for w in words}                # set
gen     = (x*x for x in range(10))               # generator (lazy)
Pythonic

Comprehensions are faster and clearer than manual for+append loops. Use a generator expression ( ) when you don't need the whole list in memory at once.

PART 3

Functions

07*args & **kwargs

python
def f(a, b=10, *args, **kwargs):
    # a: required; b: default; args: extra positionals (tuple);
    # kwargs: extra keywords (dict)
    ...
f(1, 2, 3, 4, x=5)   # a=1, b=2, args=(3,4), kwargs={'x':5}

Use * to unpack an iterable into positional args and ** to unpack a dict into keyword args when calling.

08The Mutable Default Argument Trap

python
# BUG: the default list is created ONCE and shared across calls
def add(item, target=[]):
    target.append(item); return target
add(1)  # [1]
add(2)  # [1, 2]  <- surprise!

# FIX
def add(item, target=None):
    if target is None: target = []
    target.append(item); return target
Why

Default values are evaluated once at function-definition time, not per call. Never use a mutable default — use None and create inside.

09Closures & Decorators

A closure is a nested function that captures variables from its enclosing scope. A decorator wraps a function to add behaviour without changing it — built on closures.

python
import functools
def timed(fn):
    @functools.wraps(fn)
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        result = fn(*args, **kwargs)
        print(f"{fn.__name__} took {time.perf_counter()-start:.3f}s")
        return result
    return wrapper

@timed          # slow = timed(slow)
def slow(): ...

@functools.wraps preserves the wrapped function's name/docstring.

PART 4

Object-Oriented Python

10Classes & Dunder Methods

python
class Point:
    def __init__(self, x, y):       # constructor
        self.x, self.y = x, y
    def __repr__(self):             # dev-friendly string
        return f"Point({self.x}, {self.y})"
    def __eq__(self, other):
        return (self.x, self.y) == (other.x, other.y)

Dunder ("double underscore") methods let your objects work with built-in syntax: __init__, __repr__, __eq__, __len__, __iter__, __enter__/__exit__. self is the explicit instance reference.

Inheritance

Python supports multiple inheritance; method resolution follows the MRO (C3 linearization). Use super() to call the parent.

11Properties, Class & Static Methods

python
class Circle:
    def __init__(self, r): self._r = r
    @property                       # attribute-style getter
    def area(self): return 3.14159 * self._r ** 2
    @classmethod                    # receives the class (cls)
    def unit(cls): return cls(1)
    @staticmethod                   # no self/cls
    def describe(): return "a circle"
Class vs instance variables

A variable set on the class is shared by all instances; one set on self is per-instance. Mutating a shared mutable class variable affects everyone.

PART 5

Iteration & Resources

12Iterators & Generators

An iterator implements __next__; a generator is the easy way to make one using yield, producing values lazily and keeping state between calls.

python
def countdown(n):
    while n > 0:
        yield n        # pause here, resume on next()
        n -= 1
for x in countdown(3): print(x)   # 3 2 1  (no list built)
Why generators

They process huge or infinite streams with constant memory — you only hold one item at a time. Ideal for reading big files or pipelines.

13Context Managers

The with statement guarantees setup/teardown (like closing a file) even if an exception occurs — via __enter__/__exit__.

python
with open("data.txt") as f:
    data = f.read()
# file is closed automatically here, even on error

from contextlib import contextmanager
@contextmanager
def timer():
    t = time.perf_counter(); yield; print(time.perf_counter()-t)
PART 6

Runtime, Concurrency & Memory

14The GIL & Concurrency

CPython has a Global Interpreter Lock: only one thread executes Python bytecode at a time. So threads don't give true parallelism for CPU-bound work.

WorkloadUse
I/O-bound (network, disk)threading or asyncio — the GIL releases during I/O waits.
CPU-bound (computation)multiprocessing — separate processes sidestep the GIL.
Say this

"Threads for I/O-bound, processes for CPU-bound, because of the GIL." (Newer CPython is working toward an optional GIL-free build.)

15Memory & Garbage Collection

CPython manages memory with reference counting: an object is freed when its refcount hits zero. A separate cyclic garbage collector handles reference cycles that counting alone can't.

Practical notes

Avoid unintended references that keep objects alive (e.g. large objects captured in closures or global caches). __slots__ can cut per-instance memory for classes with many instances.

PART 7

Quick-Fire Q&A

Tap a question to reveal the answer.

>Is Python compiled or interpreted?

Both: source is compiled to bytecode, then interpreted by the Python Virtual Machine (CPython).

>Mutable vs immutable types?

Immutable: int, float, str, tuple, frozenset. Mutable: list, dict, set. Immutables are hashable and safe to share.

>is vs ==?

== compares values; is compares identity (same object). Use is only for None and other singletons.

>What is the mutable default argument trap?

A default like def f(x=[]) is created once and shared across calls, accumulating state. Use None and create inside.

>What is a decorator?

A callable that wraps a function to add behaviour, using closures; applied with @decorator.

>What is a generator?

A function using yield that produces values lazily and keeps state between calls, using constant memory for large streams.

>Explain the GIL.

The Global Interpreter Lock lets only one thread run Python bytecode at a time in CPython, so use multiprocessing for CPU-bound work and threads/asyncio for I/O-bound.

>How does Python manage memory?

Reference counting frees objects at zero references, plus a cyclic garbage collector for reference cycles.

>*args vs **kwargs?

*args collects extra positional arguments as a tuple; **kwargs collects extra keyword arguments as a dict.

>list vs tuple?

Lists are mutable and used for homogeneous, growing collections; tuples are immutable, hashable, and good for fixed records.

>What does the with statement do?

It uses a context manager (__enter__/__exit__) to guarantee setup/cleanup, like closing a file even on error.

>classmethod vs staticmethod?

A classmethod receives the class (cls) and can build instances; a staticmethod receives neither self nor cls — just a namespaced function.
