PART 1

Fundamentals

01How CSS Works

CSS (Cascading Style Sheets) controls presentation. A rule is a selector plus a block of declarations (property: value).

css
.btn {
  color: white;
  padding: 8px 16px;
}

Three ways to include it: external (<link>, best), internal (<style>), and inline (style="", avoid).

02Selectors

SelectorMatches
.classElements with that class
#idThe element with that id
div pDescendant: p inside div
div > pDirect child
a:hoverPseudo-class state
[type="text"]Attribute selector

03Specificity & the Cascade

When rules conflict, the browser picks the winner by specificity, then source order.

inline style = 1,0,0,0 #id = 0,1,0,0 .class :hover [attr] = 0,0,1,0 element ::before = 0,0,0,1
!important

!important overrides specificity — use it sparingly; it's a common source of un-maintainable CSS.

04Inheritance

Some properties inherit from parent to child by default (text-related: color, font-family, line-height); most box/layout properties (margin, border, width) do not.

Force it with keywords: inherit, initial, unset.

PART 2

The Box Model

05The Box Model

Every element is a box of four layers, from inside out:

+-------------- margin --------------+ | +---------- border ----------+ | | | +------ padding ------+ | | | | | content | | | | | +---------------------+ | | | +----------------------------+ | +------------------------------------+
Margin collapse

Vertical margins between adjacent block elements collapse into the larger of the two — not their sum.

06box-sizing & display

css
/* width includes padding + border -- almost always what you want */
* { box-sizing: border-box; }

Default content-box means width is content only, so padding/border get added, breaking layouts. border-box fixes this.

display: block, inline, inline-block, flex, grid, none (removes from layout).

PART 3

Layout

07Flexbox (1-D layout)

css
.row {
  display: flex;
  justify-content: space-between; /* main axis */
  align-items: center;         /* cross axis */
  gap: 12px;
}

Flexbox lays out items in one dimension (a row or column). Key child property: flex: 1 to grow and share space.

08Grid (2-D layout)

css
.grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 16px;
}
Flex vs Grid

Use Flexbox for a single row/column of content that flows; use Grid for two-dimensional page layouts with rows and columns. fr = a fraction of free space.

09Positioning

ValueBehaviour
staticDefault; normal flow.

Responsive Design

11Media Queries (mobile-first)

css
/* base styles = mobile */
.card { width: 100%; }

/* enhance for larger screens */
@media (min-width: 768px) {
  .card { width: 50%; }
}
Mobile-first

Write base styles for small screens, then layer on min-width queries. Simpler than overriding desktop styles downward.

PART 5

Visual & Modern

12Pseudo-classes & Elements

  • Pseudo-classes (single colon) target a state: :hover, :focus, :nth-child(2), :first-child, :not().
  • Pseudo-elements (double colon) target a sub-part: ::before, ::after, ::placeholder.

13Custom Properties & Motion

css
:root { --accent: #0e7c86; }
.btn {
  background: var(--accent);
  transition: background .2s ease;
}

CSS variables (--name / var()) cascade and can change at runtime. Use transition for state changes and @keyframes + animation for sequences.

Perf

Animate transform and opacity — they're GPU-accelerated and don't trigger layout reflow.

PART 6

Quick-Fire Q&A

Tap a question to reveal the answer.

>What does the C in CSS stand for, and why does it matter?

Cascading. Multiple rules can target one element; the cascade decides the winner using specificity, importance, and source order.

>How is specificity calculated?

Inline (1000) > id (100) > class/attribute/pseudo-class (10) > element/pseudo-element (1). Higher total wins; ties break by source order.

>What is box-sizing: border-box?

It makes an element's width/height include padding and border, so boxes size predictably. Most projects set it globally.

>Flexbox vs Grid?

Flexbox is one-dimensional (a row or column); Grid is two-dimensional (rows and columns). Use Grid for page layout, Flexbox for component alignment.

>position: absolute vs fixed?

Absolute positions relative to the nearest positioned ancestor and scrolls with content; fixed positions relative to the viewport and stays put.

>em vs rem?

em is relative to the element's/parent's font-size and can compound; rem is relative to the root, so it's predictable.

>What is margin collapsing?

Adjacent vertical margins between block elements merge into the larger single margin rather than adding together.

>How do you make a site responsive?

Fluid units and layouts (%, fr, flex/grid), the viewport meta tag, and mobile-first media queries with min-width.

>Which properties are cheapest to animate?

transform and opacity — they avoid layout/paint and run on the compositor.

Final tips for the interview

  1. Be fluent in specificity and the cascade — it's the #1 CSS question.
  2. Explain the box model and why border-box helps.
  3. Choose Flexbox vs Grid deliberately and say why.
  4. Default to mobile-first, rem-based, responsive thinking.
  5. Know positioning contexts cold, especially absolute vs fixed vs sticky.
  6. Mention performance: animate transform/opacity, avoid layout thrash.
